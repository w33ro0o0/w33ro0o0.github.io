Theorie

Entstehung

1980 Der bei CERN angestellte Physiker Tim Berners-Lee entwickelt das Hypertext Programm ENQUIRE um es den Mitarbeitern zu ermöglichen Dokumente zu teilen.
1989 Berners-Lee schlägt in einem Memo ein internetbasiertes Hypertext-System vor.
1990 Berners-Lee spezifiziert HTML und entwickelt Browser- sowie Server-Software dazu.
1991 Die este öffentlich verfügbare Beschreibung von HTML in Form eines Dokuments namens "HTML Tags" wird von Berners-Lee herausgebracht. Sie enthielt 18 Elemente die das initiale Design von HTML beschrieben.
1993 Der erste Vorschlag für eine HTML Spezifikation von Berners-Lee und Dan Connolly in einem Entwurf namens "Hypertext Markup Language (HTML)" veröffentlicht.
1994 Die Internet Engineering Task Force (IETF) gründete eine HTML Arbeitsgruppe.
1995 Mit "HTML 2.0" wird von der IETF Arbeitsgruppe die erste HTML Spezifikation veröffentlicht, die als Standard für zukünftige Implementationen dienen sollte.
1996 Das World Wide Web Consortium (W3C) übernimmt die Entwicklung und Standardisierung von HTML.
2000 HTML wird ein internationaler standard (ISO/IEC 15445:2000)

Versionen

Version Datum Beschreibung

HTML5 10/2014 Schafft auf Basis von HTML 4.01 und XHTML 1.0 ein neues Vokabular. Die zu HTML gehörende DOM-Spezifikation wurde ebenfalls überarbeitet und erweitert.
HTML 5.1 11/2016 Aktuelle Version.

Neuheiten

Dokumenttypangabe

HTML5-Dokumente beinhalten eine Dokumententypangabe in Form der Zeichenkette <!DOCTYPE html> um ein standardkonformes Rendering im Browser auszulösen.
Auf Versionierung wurde bewusst verzichtet, da HTML5 Rückwärtskompatibel ist und alle Elemente aus HTML 4.01 enthält.

Einbindung von SVG und MathML im Quelltext

  • Scalable Vector Graphics (SVG) ist eine Spezifikation zur Beschreibung zweidimensionaler Vektorgrafiken.
  • Mathematical Markup Language (MathML) ist ein Dokumentenformat zur Darstellung mathematischer Formeln.

Strukturierende Elemente

figure, figcaption Auszeichnung zusätzlicher Inhalte, beispielsweise Abbildungen mit Bildunterschriften

Zusätzlich zu den div Elementen, gibt es sechs neue Elemente, die zusätzlich die Art ihres Inhaltes definieren:

nav Definiert Navigationslinks (Menü)
section Definiert eine Sektion in einem Dokument
article Definiert einen Artikel in einem Dokument
aside Definiert Inhalt abseits des Seiteninhaltes
header Definiert den Kopfbereich einer Seite/Sektion
footer Definiert den Fussbereich einer Seite/Sektion

Elemente zur Textauszeichnung

time Definiert ein Datum / eine Zeit für dynamische Lokalisierung
wbr Definiert eine mögliche Textumbruchsstelle in langen Wörtern
ruby, rp, rt Definiert Ruby-Annotierungen zwecks Darstellung ostasiatischer Zeichen mit Hinweisen zur Aussprache
mark Definiert hervorgehobenen Text

Multimedia-Elemente

audio, embed, source, track, video Einbinden von Audio- und Video-Dateien erstmals ohne zusätzliches Plugin.
canvas Definiert eine Zeichenfläche, auf der mittels Sktipten zweidimensionale Bilder gezeichnet werden können.

Formularelemente

progress Bildet den Fortschritt eines Prozesses ab
meter Definiert einen messbaren Wert innerhalb eines gewissen Wertebereichs
datalist Spezifiziert eine Liste an vordefinierten Optionen für Eingabesteuerungen (beispielsweise Vervollständigungsvorschläge)
keygen Definiert ein Feld zur Generierung eines Schlüsselpaares für Folrmulare (forms)
output Definiert das Ergebnis einer Berechnung

Das Input Element wurde erweitert:

  • Um neue Typen z. B. zur Eingabe von Suchbegriffen, Telefonnummern, URL- und E-Mail-Adressen, Datums- und Zeitangaben, Zahlen sowie Farbangaben
  • Um neue Attribute z. B. zur automatischen Vervollständigung, automatische Fokusierung, Festlegung von Minima und Maxima, Textmustervalidierung sowie Platzhalter-Werte

Interaktive Elemente

details, summary Der Inhalt des summary-Elements wird ständig angezeigt, der restliche Inhalt des details-Elements kann ein- und ausgeblendet werden.
menu, command Das menu-Element dient als Struktur für einzelne command-Elemente als Interaktionspunkt in Werkzeugleisten und (Kontext-)Menüs.

Neue API's (Application Programming Interfaces)

HTML5 definiert einige neue Schnittstellen, die mittels JavaScript genutzt werden können um Webanwendungen zu erstellen:

HTML Geolocation Bestimmung der geographischen Position des Users
HTML Drag and Drop Ziehen und Einfügen an einem anderen Ort von HTML5 Elementen
HTML Local Storage Lokale Speicherung von Benutzerdaten als Alternative zu Cookies
HTML Application Cache Zwischenspeichern von Daten im Browser für Offline-Zugriff, Geschwindigkeit und reduzierte Serverlast
HTML Web Workers Ausführen von JavaScript im Hintergrund, ohne die Performance der Seite zu beeinflussen
HTML Server-Sent Events Updates der Seite vom Server aus

Tutorial

Basics

<html> <head> <title>Basiscs</title> </head> <body> <h1>Gößte Überschrift</h1> <p>Das ist<br>ein Paragraph<br>mit Zeilenumbrüchen.</p> <hr> <h6>Kleinste Überschrift</h6> <pre> Das ist ein vorformatierter Paragraph, der Leerzeichen und Leerzeilen beinhaltet, die so wie im Source auch im Browser angezeigt werden. </pre> </body> </html>
<html> Definiert die Wurzel eines HTML Dokuments
<body> Container für den sichtbaren Teil des HTML Dokuments
<head> Container für Kopf-Elemente, wie Titel, Skripts, Styles oder Meta-Informationen
<tile> Definiert den Titel des HTML Dokuments im Kopfteil
<h1> - <h6> Definiert Überschriften
<hr> Definiert eine thematische Änderung des Inhalts
<p> Definiert einen Paragraphen
<br> Fügt einen einzelnen Zeilenumbruch ein
<pre> Definiert einen vorformatierten Text

Styles

style Attribut um den Style von HTML-Elementen mit folgenden Eigenschaften festzulegen
background-color Hintergrundfarbe
color Textfarbe
font-family Schriftart
font-size Schriftgröße
text-align Textausrichtung
